Un herrero le asigna una tarea a su aprendiz que consiste en hacer una selección de anillos (aros). El aprendiz no es aún experto en este arte y como resultado, algunos (para ser honesto, la mayoría) de los anillos terminan conectados entre sí. Ahora él está pidiendo tu ayuda separar los anillos y básicamente, para decidir cómo romper suficientes anillos para liberar el máximo número de anillos posibles.

Todos los anillos están numerados y se te informará cuales están conectados. Esta información se provee como una secuencia de sets, donde cada set describe los anillos conectados. Por ejemplo: {1, 2} significa que el primero y segundo anillos están conectados. Deberás contar cuántos anillos deberán romperse para conseguir el máximo número de anillos separados. Los anillos están numerados desde 1 hasta N, donde N es la cantidad total de los anillos.

En la imagen de arriba puedes ver las conexiones: ({1,2}, {2,3}, {3,4}, {4,5}, {4,6}, {6,5}). La mejor solución en este caso sería romper 3 anillos, obteniendo así 3 anillos completos y separados. Para este ejemplo, el resultado es 3.

Datos de Entrada: Información sobre los anillos conectados, como una tupla de sets con enteros (int).

Salida: El número de anillos a romper, como un entero (int).
